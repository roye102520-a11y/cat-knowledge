Cat Knowledge · 静态站使用说明
===============================

本文件夹为「静态网站」导出结果，需通过 HTTP 访问（勿直接用 file:// 打开 index.html）。

本地预览示例：
  npx serve . -p 3330
然后在浏览器打开 http://localhost:3330

或将本目录内容上传到 GitHub Pages、云存储静态托管等。

数据已打包在页面脚本中，无需服务器与数据库。

站内包含：
  · 问题 / 用品 / 指南 / 搜索 / AI 问答（检索均在本机页面内完成）
  · 科普长文：/articles/（中英双语 SEO 长文，与 content/seo-articles 源文件对应）

重新生成静态站：在源码目录执行（该目录内须有 package.json；文件夹名一般为 cat-knowledge-base）：
  cd /你的路径/cat-knowledge-base
  npm run build:share
  若当前目录是「.env.local」等上级文件夹，直接 npm 会报错 ENOENT / 找不到 package.json。


首次把源码同步到 GitHub（在 cat-knowledge-base 目录内）
------------------------------------------------
  1. git init
  2. git add .
  3. git commit -m "Initial commit"
  4. 打开 github.com → New repository → 新建空仓库（可不勾选 README，减少冲突）
  5. git remote add origin https://github.com/你的用户名/仓库名.git
  6. git branch -M main
  7. git push -u origin main

以后有改动：
  git add .
  git commit -m "简要说明本次修改"
  git push

若已按下文开启 GitHub Pages（Actions 部署），每次 push 到 main 会自动重新发布网站。
注意：勿把密码、Notion Token 等写入仓库；.env.local 已在 .gitignore 中默认忽略。


免费「可点击」网页（任选其一，均无需付费）
==============================

A) GitHub Pages（推荐，稳定）
  1. 在 github.com 新建仓库，把本项目 push 到 main 分支（需含 .github/workflows 与源码）。
  2. 仓库 Settings → Pages → Build and deployment → Source 选择 「GitHub Actions」。
  3. 再点一次 Actions 页，等 「Deploy static site to GitHub Pages」 跑绿。
  4. 打开：https://你的用户名.github.io/仓库名/
     若仓库名恰好是 「用户名.github.io」，则根地址为 https://用户名.github.io/

B) Netlify Drop（最快，不用 Git）
  1. 本地执行：npm run build:share
  2. 打开 https://app.netlify.com/drop ，把「out」文件夹拖进去即可获得免费 https 链接。

C) Surge（命令行一键）
  1. npm run build:share
  2. cd out && npx surge . （按提示注册/登录即可得到 yourname.surge.sh）

说明：本地 zip 分享请不要设置环境变量 BASE_PATH；仅 GitHub Actions 构建时会自动加子路径。
